http缓存
1. 强缓存
  不会向服务器发请求，直接冲缓存中读取资源
  cache-control: 存活的时间，最大生命值
  Expires: 对照本地时间，存活的时间

2. 协商缓存
  Last-Modified: 服务器告诉浏览器最后修改的时间
  Etag: 服务器响应请求时，生成唯一的标识符
  浏览器发起请求数据时，这两者与服务器上的进行对比
  不同: 返回新数据和标识符
  相同：返回304，从缓存中提取信息

3. 优点
  减少了冗余的数据传递，节省宽带流量
  减少了服务器的负担，大大提高了网站性能
  加快了客户端加载网页的速度 这也正是HTTP缓存属于客户端缓存的原因。
